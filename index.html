    We were asked to come up with 3 differences between Web 1.0, Web 2.0 and Web 3.0:

    Criteria\Generation Web 1.0 Web 2.0 Web 3.0
    Focus Companies: Web 1.0 gave Companies a platform where they could express themselves. It is essentially like a restaurant's sign or a menu. Communities: With Web 2.0, interactive sites started popping out, eventually leading to sites that are pure "platforms" - they only take care of the infrastructure behind the site, the content is made completely by the users. Individuals: Web 3.0 took the progress made in AI and balanced its reliance on the herd (which is a characteristic of Web 2.0) with heuristics and predictions based on various parametres in order to understand the implicit meaning behind the user.
    Possibilities Read-Only: All web pages based on Web 1.0 are static, meaning only the administrator has the authority to alter the content of the site. Visitors could only see what the admin had posted and could not react to it in any way whatsoever. Read-Write: Web 2.0 bestowed the right of editing upon the regular user, meaning the internet is no longer an amorphic encyclopedia, it is a network, where one can see and publish content, and where one's actions trigger dynamic events. Personalisation: The evolution of Artificial Intelligence and NLP in the 21th century allowed the web to comprehend the meaning of words, rather than relying on naive .equals() between index values and queries. The web tries to guess what you want to see with both sides not completely sure what are they looking for.
    Advertising Banners & Views: In the age of Web 1.0, where user interactions were not even a thing, exposure is what mattered the most. The focus as a content creator was on getting users to visit your site, and, as a advertiser, to make your banner as eye-catching as you possibly can. CPC: With the support for user interactions in Web 2.0, ads evolved into a model where exposure to an ad (Cost per mille) is far less effective than clicking on it (Cost per click) and getting redirected the advertiser's site. This changed the game by adding the element of clicks (misclicks, fraud clicks, malware etc.) Engagement: Web 3.0 added a new layer onto Web 2.0: The rise of Recommendation Systems was quickly harnessed for ad personalisation, meaning different users see different ads based on numerous parametres. data is gathered on the user in order to build a reliable relevance feedback, way more complex data than "Did the user click the ad? Y/N"


    Now, We shall explore the difference between the usability through use cases:



    Searching "Side effects of Acamol drug" in different eras of the web
    Web 1.0 Web 2.0 Web 3.0
    A user would typically need to have prior knowledge about sites who specialise in summarising side effects of pharmaceuticals, or rather, have extensive knowledge in writing efficient and precise queries in old search engines like altavista to find his answer. A good example would be looking up "side effects" under the definition of Acamol in a pharmaceutical database. The user can go to google and ask a search engine, who can then answer him based on pre-processing the web beforehand and retrieving relevant web documents rapidly. He can also turn to social media and ask his friends. For example, Google, Facebook, Twitter, Reddit, 4chan, Tumblr, and Wikipedia are good examples for use cases in Web 2.0 In Web 3.0, you can ask Google, who already has tons of data on you, and get accurate answers based on billions of similar queries and their relevance feedbacks. Some examples for services which can be considered Web 3.0 are: Google (as they ditched old, flawed models like PageRank and moved towards word embedding models like Word2Vec), Virtual assistants (like Cortana and Siri) and Modern E-Commerce companies (like Amazon, which heavily utilises Recommendation Systems).
